# Протокол воркшопа — изоляция измерений от шума

Как участники выполняют каждый этап, чтобы `ccusage` фиксировал **только** токены
целевого прогона, а не побочную активность.

> **Связанные документы:** [charter.md](./charter.md) — архитектура;
> [agenda.md](./agenda.md) — run-sheet фасилитатора.

---

## Проблема

`ccusage` измеряет **всю** активность Claude Code в проекте. Если участник правит
AGENTS.md, строит прокси и решает задачу вперемешку — замеры неинформативны.

## Решение: снапшоты вокруг одной сессии

`workshop:run*` делает снапшот ДО того как участник откроет Claude Code, и ПОСЛЕ
того как закроет. Всё что между — измеряется. Всё что вне — нет.

| Активность | Инструмент | Измеряется? |
|---|---|---|
| **Прогон задачи** | Интерактивный Claude Code (участник сам пишет промпт) | ✅ ДА — между снапшотами |
| **Оптимизация** (AGENTS.md, прокси, хуки) | Интерактивный Claude Code или редактор | ❌ НЕТ — вне окна снапшотов |

---

### Настройка агента (pre-flight)

| Агент | Измерение | Что настроить до воркшопа |
|---|---|---|
| **Claude Code** | `ccusage` авто | Ничего |
| **Codex** | `.codex-log/` в корне репо | запуск из `apps/angular-demo/` с `-c log_dir=../.codex-log` (или абсолютный `log_dir` в `~/.codex/config.toml`) |
| **Cursor** | Gate-only | Ничего |

## Пошаговый протокол

### Run 1 — Раздутый Baseline

```bash
npm run workshop:run1
```

Скрипт делает:
1. `reset` — сбрасывает код к buggy baseline
2. 📸 снапшот ДО
3. Показывает: «cd apps/angular-demo, открой агента, реши задачу, закрой агента»
4. Ждёт Enter
5. 📸 снапшот ПОСЛЕ
6. Вычисляет дельту
7. `scenario:verify`
8. Сохраняет в `.workshop/run-run1.json`

Участник в это время: переходит в `apps/angular-demo/`, открывает агента, пишет промпт,
агент работает, участник закрывает агента, жмёт Enter.

### Оптимизация AGENTS.md (между Run 1 и Run 2)

**Не измеряется.** Участник открывает AGENTS.md (в редакторе или через Claude Code)
и применяет рычаги из `hygiene-methods.md`. Это может быть отдельная Claude Code
сессия — она не попадёт в замер потому что `workshop:run2` сделает новый снапшот ДО.

### Run 2 — Гигиена

```bash
npm run workshop:run2
```

Всё то же самое. Участник снова открывает Claude Code и решает ту же задачу — но
теперь с оптимизированным AGENTS.md. Возможно, с более точным промптом.

Скрипт показывает сравнение с Run 1.

### Оптимизация инструментов (между Run 2 и Run 3)

**Не измеряется.** Участник выбирает Option A (прокси) или Option B (хуки) и строит
решение. Можно использовать Claude Code.

### Run 3 — Уровень инструментов

```bash
npm run workshop:run3
```

Скрипт делает замер + сравнение с Run 1 и Run 2 + общий прогресс.

---

## Что делает `workshop:run*` (внутреннее устройство)

```bash
# Эквивалент ручных команд:
npm run scenario:reset
# ← скрипт делает снапшот ДО
# ← участник открывает Claude Code, решает задачу, закрывает
# ← скрипт делает снапшот ПОСЛЕ и вычисляет дельту
npm run scenario:verify
# ← результат сохраняется в .workshop/
```
